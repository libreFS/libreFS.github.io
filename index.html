Why does libreFS exist?

MinIO built a great open-source object storage server. Then, in May 2025, MinIO Inc. began moving its most valuable features — the embedded WebUI, LDAP & OIDC login, distributed erasure coding, and pre-built binaries — into AIStor, their proprietary commercial product.

The last fully open-source MinIO release was April 22, 2025. libreFS is forked from that exact commit, maintained by the community to keep all those features free and open, permanently.

  • Apr 2025 MinIO's last fully open-source release (RELEASE.2025-04-22)
  • May 2025 MinIO moves WebUI, LDAP/OIDC & distributed mode to AIStor
  • Apr 2026 libreFS v1 — first community release, all features preserved

Not a protest — a commitment

libreFS isn't just a snapshot. We actively cherry-pick security fixes and bug fixes from upstream, maintain binary releases for all platforms, and keep the project production-ready for the long term.

Drop-in replacement

Same S3 API. Same environment variables. Same admin API. If you were running MinIO, you can switch to libreFS with zero application changes — no migration, no reconfiguration.

Community governed, no telemetry

libreFS will never phone home, collect usage data, or add external network calls. Every release is audited. Governance is open — no single company controls the roadmap.

Everything MinIO used to offer — kept free

These are the features MinIO moved to AIStor. In libreFS, they are open-source and always will be.

Full Embedded WebUI

The complete management console on port 9001. Browse buckets, manage users, configure policies — all from your browser, no external service required.

LDAP & OIDC Authentication

Authenticate users against any LDAP directory or OIDC-compatible identity provider — Keycloak, Dex, Okta, Google, Azure AD, and more. Includes group-based access policies.

Distributed Erasure Coding

Multi-node deployments with erasure coding for high availability and fault tolerance. Survive drive and node failures without losing data or availability.

Pre-built Binaries

Download and run — no Go toolchain required. Signed binaries for Linux, macOS, and Windows on both amd64 and arm64, plus Docker images on GHCR.

100% S3 API Compatible

Every S3 operation works as expected. All existing clients, SDKs, and tools built for AWS S3 or MinIO work with libreFS without any changes to your application code.

Encryption & KMS

Server-side encryption with KES/KMS integration. SSE-S3, SSE-C, and SSE-KMS supported. Integrates with HashiCorp Vault, AWS KMS, Azure Key Vault, and more.

Quick Start

Up and running in under a minute. Pick your preferred method.

# Start libreFS with Docker — WebUI available at http://localhost:9001
docker run -d \
  -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=admin \
  -e MINIO_ROOT_PASSWORD=password \
  -v /tmp/data:/data \
  ghcr.io/librefs/librefs:latest server /data \
  --console-address :9001

S3 API on :9000 · Web console on :9001 · All versions and platforms 

# Linux amd64 — verify checksum before running
curl -LO https://github.com/libreFS/libreFS/releases/latest/download/librefs-linux-amd64
curl -LO https://github.com/libreFS/libreFS/releases/latest/download/librefs-linux-amd64.sha256sum
sha256sum -c librefs-linux-amd64.sha256sum

chmod +x librefs-linux-amd64
./librefs-linux-amd64 server /tmp/data --console-address :9001

Builds for linux/amd64, linux/arm64, darwin/amd64, darwin/arm64, windows/amd64 · All downloads 

# Requires Go 1.24+
git clone https://github.com/libreFS/libreFS.git
cd libreFS
go build -o librefs .
./librefs server /tmp/data --console-address :9001

Default credentials: MINIO_ROOT_USER=minioadmin / MINIO_ROOT_PASSWORD=minioadmin 

# Install lc — the libreFS CLI client (drop-in replacement for mc)
curl -LO https://github.com/libreFS/librefs-cli/releases/latest/download/lc-linux-amd64
chmod +x lc-linux-amd64&& sudo mv lc-linux-amd64 /usr/local/bin/lc

# Register your server and run basic operations
lc alias set local http://localhost:9000 admin password
lc mb local/my-bucket
lc cp ./myfile.txt local/my-bucket/
lc ls local/my-bucket

lc is a fork of MinIO's mc — every mc command works as lc. Download lc for all platforms
