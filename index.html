Why does libreFS exist?

MinIO built a great open-source object storage server. Then, in May 2025, MinIO Inc. began moving its most valuable features — the embedded WebUI, LDAP & OIDC login, distributed erasure coding, and pre-built binaries — into AIStor, their proprietary commercial product.

The last fully open-source MinIO release was April 22, 2025. libreFS is forked from that exact commit, maintained by the community to keep all those features free and open under AGPL-3.0 — permanently.

  • Apr 2025 MinIO's last fully open-source release (RELEASE.2025-04-22)
  • May 2025 MinIO moves WebUI, LDAP/OIDC & distributed mode to AIStor
  • Apr 2026 libreFS v1 — first community release, all features preserved

Not a protest — a commitment

libreFS isn't just a snapshot. We actively cherry-pick security fixes and bug fixes from upstream, maintain binary releases, and keep the project production-ready for the long term.

Drop-in replacement

Same S3 API. Same environment variables. Same admin API. If you were running MinIO, you can switch to libreFS with zero application changes.

No telemetry, ever

libreFS will never phone home, collect usage data, or add any call to external endpoints without explicit user opt-in. Audited on every commit.

Everything MinIO used to offer — kept free

These are the features MinIO moved to AIStor. In libreFS, they are open-source and always will be.

🖥

Full Embedded WebUI

The complete management console on port 9001. Browse buckets, manage users, set policies — all from your browser, no external service needed.

🔐

LDAP & OIDC Login

Authenticate WebUI and S3 users against any LDAP directory or OIDC-compatible identity provider (Keycloak, Dex, Okta, Google, and more).

🗄

Distributed Erasure Coding

Multi-node deployments with erasure coding for high availability and fault tolerance. Scale from a single node up to a large cluster.

📦

Pre-built Binaries

Download and run — no Go toolchain required. Binaries for Linux, macOS, and Windows on both amd64 and arm64, plus Docker images on GHCR.

🔌

100% S3 Compatible

Every S3 API operation works exactly as expected. All existing clients, SDKs, and tools that work with AWS S3 or MinIO work unchanged with libreFS.

🔒

Encryption & KMS

Server-side encryption with KES/KMS integration. SSE-S3, SSE-C, and SSE-KMS. Your data at rest is always encrypted if you need it to be.

Quick Start

Running libreFS takes under a minute. Pick your preferred method below.

# Pull and run — open http://localhost:9001 when ready
docker run -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=admin \
  -e MINIO_ROOT_PASSWORD=password \
  -v /tmp/librefs-data:/data \
  ghcr.io/librefs/librefs:latest server /data \
  --console-address :9001

Opens the S3 API on :9000 and the web console on :9001.
See Downloads for specific version tags and all available images.

# Download the latest binary (Linux amd64 example)
curl -LO https://github.com/libreFS/libreFS/releases/latest/download/librefs-linux-amd64
chmod +x librefs-linux-amd64

# Verify the checksum
curl -LO https://github.com/libreFS/libreFS/releases/latest/download/librefs-linux-amd64.sha256sum
sha256sum -c librefs-linux-amd64.sha256sum

# Start the server
./librefs-linux-amd64 server /tmp/data --console-address :9001

Binaries for linux/amd64, linux/arm64, darwin/amd64, darwin/arm64, windows/amd64. See the Downloads page for all platforms and direct links.

# Requires Go 1.24+
git clone https://github.com/libreFS/libreFS.git
cd libreFS
go build -o librefs .
./librefs server /tmp/data --console-address :9001

Default credentials: MINIO_ROOT_USER=minioadmin / MINIO_ROOT_PASSWORD=minioadmin.
Set those env vars before starting to use your own credentials.

# Download lc — the libreFS CLI client
curl -LO https://github.com/libreFS/librefs-cli/releases/latest/download/lc-linux-amd64
chmod +x lc-linux-amd64 && sudo mv lc-linux-amd64 /usr/local/bin/lc

# Connect to your libreFS instance
lc alias set local http://localhost:9000 admin password

# Common operations
lc admin info local
lc mb local/my-bucket
lc cp ~/myfile.txt local/my-bucket/
lc ls local/my-bucket

lc is the libreFS CLI client, a fork of MinIO's mc. It is fully compatible with all mc commands — any mc command works as lc. See the Downloads page for all platforms.
